    HotKeys for PBCC program?

    12 Mar 2022, 07:18 PM
    Maybe the dumbest question to date?
    >>> Is it possible for a PBCC program to be loaded and remain dormant until activated via HotKey?

    I've done this in PBWin numerous times, but the setup functions require a handle to a window, and this program would not have or open a window at all.
    The program I'm working on simply reformats (in specific ways) a clipboard text, silently, so that the user can then paste it into whatever...

    I recall that PBDOS had some kind of "popup" commands, but those memories are probably not relevant, although the concept is the same.

    In PBWin, my calls to RegisterHotKey are in the callback function of the main window, so I can access CB.msg to determine states such as
    CASE %WM_INITDIALOG (where I obtain a value from GlobalAddAtom and then do RegisterHotKey)
    CASE %WM_HOTKEY (where I check SELECT CASE CB.WParam for a value equal to my constant for that function, then run the relevant formatting code).

    So without PBWin's structure, is it possible for PBCC to have the equivalent of a CALLBACK function, and if yes, what kind of structure would that be?

    Please note that this is purely an academic question... By the time anyone reads this, I'll probably have it coded in PBWin, but I won't be able to sleep well until my curiosity is addressed.

    Thanks,
    -John

        12 Mar 2022, 11:39 PM
        this program would not have or open a window at all.
        Or you could use a console-free app that uses PeekMessage in a loop?..
        '
        Code:
        #COMPILE EXE
#CONSOLE OFF
#DIM ALL
#INCLUDE "win32api.inc"
 
'------------------/
 
FUNCTION PBMAIN () AS LONG
 LOCAL nAtom AS DWORD, COUNT, lRes AS LONG
 
  nAtom = GlobalAddAtom("Unique Hotkey")
  RegisterHotKey 0, nAtom, %MOD_ALT OR %MOD_CONTROL, ASC("Q") ' Define 'Ctrl+alt+Q' HotKey
  WinBeep 1800,200
 '  MessageBox 0, "Ctrl+alt+Q", Str$(nAtom), %MB_SYSTEMMODAL OR %MB_ICONMASK
  DO                                                ' Main Program Loop
    SLEEP 100  :  INCR COUNT
   LOCAL tmsg AS TagMsg
    PeekMessage (tmsg, %NULL, %WM_HOTKEY, %WM_HOTKEY, %PM_NoRemove)   ' message loop
      TranslateMessage(tmsg)
 '      If tmsg.message = %WM_HOTKEY Then       ' Redundant?
        IF tmsg.wparam = nAtom THEN
          WinBeep 800,200
          SetClipTextCase()
          lRes = MessageBox(0, "End Process?", "Count"+STR$(COUNT), %MB_SYSTEMMODAL OR %MB_DEFBUTTON2 OR %MB_YESNO)
            IF lRes = %IDYES THEN EXIT LOOP
            RESET tmsg, COUNT
        END IF
 '      End If
    IF COUNT > 200 THEN EXIT LOOP                   ' eventual timeout
  LOOP
  WinBeep 300,800
  UnregisterHotKey 0, nAtom                         ' clean up
  GlobalDeleteAtom nAtom
 
END FUNCTION
'------------------/PBMain
 
SUB SetCLipTextCase()
 STATIC ULCase AS LONG
 LOCAL sText AS STRING
 
  CLIPBOARD GET TEXT TO sText
    ULCase XOR= 1
    sText = IIF$(ULCase=1, UCASE$(sText), LCASE$(sText))
  CLIPBOARD SET TEXT sText
 
END SUB
'------------------/SetCLipTextCase
        (Use a clipboard viewer to observe pre-copied text..)
